📜 队列规则说明

优先级排序

P0 最高优先级 - 系统紧急任务
P1 高优先级 - 重要功能开发
P2 中优先级 - 常规任务
P3 低优先级 - 优化改进

四象限分类

Q1 重要且紧急 立即处理
Q2 重要不紧急 计划执行
Q3 不重要紧急 快速处理
Q4 不重要不紧急 最后处理

Agent分配规则

• 优先分配给 idle 状态的Agent
• 按 Agent 角色推荐：
  • 代码开发 → coder / coder-1 / coder-2
  • 深度分析 → deep
  • 快速响应 → fast
  • 沟通任务 → chat
  • 通用任务 → main
• 避免重复：相同任务不会被分配给多个Agent

综合评分算法

重要性 40%
紧急性 30%
依赖性 15%
可行性 10%
价值 5%
总分 = Σ(维度得分 × 权重)

🔄 自动调度机制

• 扫描频率：每 5 秒检测一次 pending 任务
• 调度器：agent-dispatcher 自动分配任务给空闲 Agent
• 状态更新：Agent 接收任务后状态变为 doing，完成后变为 completed
• 故障转移：Agent 离线超过阈值时，自动转移任务给其他 Agent

📊 任务状态流转

pending → doing → completed | 分支状态: failed blocked

👤 手动操作

• 点击任务：查看任务详情页面
• 自动调度：手动触发任务分配给空闲 Agent
• 启动 Agent：启动所有离线的 Agent
• 恢复 Agent：自动检测并恢复离线 Agent

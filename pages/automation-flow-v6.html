📊 节点详细说明
序号 节点名称 谁在执行 输入数据 处理逻辑 输出结果 状态
1 Subagent 启动 🤖 Gateway 创建的独立会话 POST /api/sessions/spawn
{ task, agentId } 		Gateway 内部：
1. 加载 Agent 配置（model, systemPrompt, tools）
2. 创建独立会话
3. 注入任务数据
4. Agent 开始自主执行 		Subagent 会话开始运行 ✅ 正常
2 读取 breakdown 🤖 Subagent task.breakdown 字段 breakdown 结构：
[{ step: 1, title: "需求分析", status: "pending" }, ...]
按顺序读取每个步骤 		步骤列表 ✅ 正常
3 逐步执行步骤 🤖 Subagent 步骤数据 执行循环：
1. 更新步骤状态 = "running"
2. Agent 自主执行步骤内容
3. 记录执行结果
4. 更新步骤状态 = "completed"
5. 调用 API 同步
POST /api/task-execution/step 		步骤执行结果 ✅ 正常
4 auto-monitor ⚙️ PM2 后台进程 tasks.json 中 status="doing" 的任务 每 5 分钟检查：
1. 任务推进率：doing 任务是否有活动
2. 超时检测：超过 30 分钟无更新
3. Agent 活跃度：Agent 是否在线
4. 异常时触发 auto-recovery 		监控报告 + 异常告警 ✅ online
5 auto-recovery ⚙️ PM2 后台进程 卡住的任务 恢复策略：
1. 自动重试：重新启动 Subagent（最多3次）
2. 降级处理：标记需人工处理，发通知
3. 重新分配：换其他 Agent 执行 		任务恢复或告警 ✅ online

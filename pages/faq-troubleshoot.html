快速入门

? 如何创建第一个任务？

创建任务有以下几种方式：

  1. 自然语言创建：直接在聊天中描述任务，如"帮我整理周报"、"调研WSUS"
  2. 手动创建：访问任务管理页面，点击"创建任务"按钮
  3. API创建：调用 POST /api/tasks 接口

系统会自动分析任务意图并创建相应的任务记录。

? 系统支持哪些Agent？

系统内置以下Agent：

Agent用途模型
main主控AgentGLM-5
deep深度思考DeepSeek-V3.2
chat对话交互GLM-4.7
fast快速响应Kimi-K2.5
office办公任务MiniMax-M2.5
? 如何访问系统？

系统通过以下方式访问：

  • Web界面：访问 http://localhost:8081
  • 飞书：通过飞书机器人发送消息
  • API：调用 RESTful API 接口

主要页面入口：

  • 任务看板：/tasks-kanban.html
  • 任务列表：/tasks-v2.html
  • 仪表盘：/dashboard-new.html
  • Agent管理：/agents-new.html

任务问题

? 任务一直处于"待执行"状态怎么办？

可能原因：

  1. 自动执行器未启动
  2. 所有Agent都在忙碌
  3. 任务被暂停

解决方案：

  1. 检查自动执行器状态：curl http://localhost:8081/api/auto-executor/status
  2. 查看任务队列：访问 /execution-queue.html
  3. 手动开始任务：调用 POST /api/tasks/:id/start
  4. 重启执行器：pm2 restart auto-task-executor
? 任务执行失败如何处理？

处理步骤：

  1. 查看任务详情，了解失败原因
  2. 检查任务步骤中的错误信息
  3. 根据错误类型选择：
    • 参数错误：修改任务参数后重试
    • 权限不足：检查API权限配置
    • 超时：增加超时时间或优化执行逻辑
  4. 使用"重试"功能或手动执行失败步骤

自动恢复：

系统每5分钟检测卡住的任务步骤，自动尝试恢复。

? 如何查看任务执行日志？

查看任务日志的方法：

  1. 任务详情页：点击任务查看详情，展开步骤查看结果
  2. 执行队列页面：/execution-queue.html 查看实时执行状态
  3. 日志文件：
    • logs/auto-executor.log - 执行器日志
    • logs/auto-monitor.log - 监控日志
    • logs/dispatcher.log - 调度日志
  4. API查询：GET /api/tasks/:id/execution

Agent问题

? Agent不响应怎么办？

排查步骤：

  1. 检查Agent状态：访问 /agents-new.html
  2. 查看Agent心跳： 
    curl http://localhost:8081/api/agents/status
  3. 检查最近一次响应时间
  4. 如果是模型超时，检查API配额

常见原因：

  • 模型API调用失败（配额耗尽、网络问题）
  • Agent进程崩溃
  • 任务堆积导致响应延迟
? 如何切换Agent使用的模型？

方法一：配置文件修改

修改 agents.list[].model.primary 配置

方法二：API动态切换

POST /api/agents/:id/model
{
  "model": "nvidia-2/deepseek-ai/deepseek-v3.2"
}

方法三：备用模型

在配置中设置 fallback_models，失败时自动切换。

系统问题

? 服务无法启动怎么办？

排查步骤：

  1. 检查端口占用：lsof -i :8081
  2. 查看错误日志：tail -50 logs/server.log
  3. 检查依赖安装：npm list
  4. 检查Node版本：node -v (需要 v18+)

常见错误：

  • MODULE_NOT_FOUND - 运行 npm install
  • EADDRINUSE - 端口被占用，杀死占用进程
  • ECONNREFUSED - 服务未启动
? 页面加载失败或显示异常？

解决方案：

  1. 清除浏览器缓存 (Ctrl+Shift+R 强制刷新)
  2. 检查浏览器控制台错误 (F12)
  3. 确认静态文件存在：ls public/*.html
  4. 检查CSS/JS加载：控制台 Network 标签

常见问题：

  • 侧边栏不显示 → 检查 js/sidebar.js 加载
  • 样式异常 → 检查 css/frame.css 加载
  • 图标不显示 → 检查 Remix Icon CDN
? Gateway频繁重启如何处理？

检查步骤：

  1. 查看重启次数：openclaw gateway status
  2. 查看错误日志：journalctl --user -u openclaw-gateway -n 50
  3. 检查配置错误（尤其是 allowAgents）
  4. 检查系统资源：CPU、内存、磁盘

常见原因：

  • 配置文件语法错误
  • API配额耗尽
  • 内存不足导致 OOM
  • 插件加载失败

API问题

? API返回401/403错误？

401 未授权：

  • 检查是否需要认证令牌
  • 确认Token未过期

403 禁止访问：

  • 检查用户权限配置
  • 确认 API 路径权限设置
  • 检查请求头 x-user-id
? API响应很慢怎么办？

排查方法：

  1. 检查网络延迟
  2. 查看API响应时间：time curl ...
  3. 检查服务器负载：top, free -h
  4. 检查数据库文件大小

优化建议：

  • 使用缓存减少重复请求
  • 分页获取数据
  • 清理历史数据

数据问题

? 如何备份和恢复数据？

手动备份：

./scripts/backup.sh [版本号]

自动备份：

系统在每次发布前自动备份。

恢复数据：

./scripts/rollback.sh [版本号] [环境]

备份位置：

  • backups/ - 代码和配置备份
  • data/ - 任务和配置数据
? 数据文件太大怎么办？

清理步骤：

  1. 查看文件大小：ls -lh data/*.json
  2. 清理历史日志：rm logs/*.log
  3. 清理完成的任务（可选）：
curl -X POST http://localhost:8081/api/tasks/cleanup \
  -H "Content-Type: application/json" \
  -d '{"status": "completed", "older_than_days": 30}'

🎯 任务系统迁移指南

任务系统迁移备份包含完整的源代码、数据、配置，可直接迁移到新环境启用。

触发迁移备份
在系统设置页面中，点击"任务系统迁移备份"的"生成迁移包"按钮。
下载迁移包
备份完成后，迁移包将保存到
/media/morning/数据/openclaw-auto-backup/task-system-migration/
解压迁移包
在目标机器上解压迁移包：
tar -xzf task-system-migration-*.tar.gz cd task-system-*/
部署文件
# 部署源代码 cp -r task-system-v2 ~/.openclaw/workspace/ cd ~/.openclaw/workspace/task-system-v2 # 安装依赖 chmod +x ../install-dependencies.sh ../install-dependencies.sh # 解压数据和配置 tar -xzf ../data-config-*.tar.gz # 解压记忆系统 mkdir -p ~/.openclaw/workspace/memory tar -xzf memory-backup/memory-*.tar.gz -C ~/.openclaw/workspace/memory/
启动服务
# 启动主服务 node server.js & # 启动核心服务（按顺序） node scripts/task-dispatcher-v2.js & node scripts/agent-queue-executor.js & node scripts/automation-monitor.js & node scripts/auto-recovery.js &
验证部署
检查服务是否正常运行：
# 检查进程 ps aux | grep "node.*task-system" | grep -v grep # 检查端口 lsof -i :8081 # 检查HTTP服务 curl http://localhost:8081/api/health
❓ 常见问题
Q: 记忆小时级备份会占用太多空间吗？
A: 不会。每个备份约500K，7天共91个文件，总大小约45M，完全在可控范围内。
Q: 如何手动触发备份？
A: 在系统设置的"系统备份"页面中，可以手动触发"记忆备份"、"任务系统迁移备份"或"完整系统备份"。
Q: 旧备份会自动清理吗？
A: 会。每天凌晨5点会自动清理超过保留期的旧备份，避免磁盘空间被填满。
Q: 如何查看备份历史？
A: 在系统设置的"系统备份"页面中，可以查看最近执行的备份记录。
Q: 停机通知与计划性停机时，可以手动备份吗？
A: 在停机通知与计划性停机后，可查看停机通知或计划性停机时，根据停机原因（如网络中断、设备故障、电源异常、计划性维护等），选择合适的手动备份时机或备份方式。关键是要在停机前后分别进行备份，在停机前先完成全量/关键数据备份，在停机后检查数据的一致性并根据需要做差异/增量备份。执行前记得仔细查看系统页面备份指南和通知要求。

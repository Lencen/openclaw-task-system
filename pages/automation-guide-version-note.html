📋 版本更新说明

v2.1 (2026-03-27)

  • 架构调整：Auto Executor 更名为 Auto Assigner
  • 职责变更：Auto Assigner 只负责检测任务和写入分配记录
  • 新增执行器：Main Agent 作为任务调度器，负责启动所有 Subagent

⚠️ 架构说明

当前架构下，只有 Main Agent 会话有 Gateway 环境，因此由 Main Agent 统一启动所有 Subagent。

这是一种集中式任务调度模式：任务虽然分配给不同 Agent，但都由 Main Agent 启动执行。

角色 职责
Auto Assigner 检测待分配任务，写入分配记录
Main Agent 启动所有 Subagent（集中调度）
Subagent (coder/deep/...) 执行分配给自己的任务

通信渠道规范与规则
1. 渠道配置规则
  • 连接模式：支持 websocket 和 webhook 两种模式
  • 认证配置：正确配置 App ID 和 App Secret
  • 权限申请：在开发者后台申请必要的 API 权限
  • 回调地址：webhook 模式需要配置公网可访问的 HTTPS 地址
  • 长连接：websocket 模式需要在开发者后台开启长连接
学习渠道配置规范
2. 消息安全规范
  • 消息验证：验证消息来源和签名
  • 敏感信息：不在消息中传递敏感信息
  • 权限隔离：不同渠道使用不同的账号配置
  • 消息去重：处理消息重复推送的情况
  • 审计日志：记录所有消息收发日志
学习消息安全规范
3. Agent 分配规则
  • 默认 Agent：为每个渠道配置默认处理 Agent
  • 私聊策略：dmPolicy 控制私聊消息的处理方式
  • 群聊策略：groupPolicy 控制群聊消息的处理方式
  • 提及要求：requireMention 控制是否需要 @机器人
  • 来源限制：allowFrom 限制消息来源
